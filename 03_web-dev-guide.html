Step 01

폴더 & 파일 구조 잡기

코딩 시작 전 가장 먼저 할 일입니다. 처음부터 구조대로 만들지 않으면 나중에 수정이 매우 어렵습니다.

my-project/ ← 프로젝트 최상위 폴더 (프로젝트명으로 변경) │ ├── index.html ← 메인 페이지 (반드시 최상위, 소문자) ├── about.html ← 서브 페이지 (기획서에 따라 추가) ├── list.html ← 목록 페이지 ├── detail.html ← 상세 페이지 ├── contact.html ← 문의/제보 페이지 │ ├── css/ │ └── common.css ← 헤더·푸터·버튼 등 공통 스타일 │ ├── js/ │ └── main.js ← jQuery 인터랙션 스크립트 │ ├── images/ │ ├── logo.png │ ├── hero-bg.jpg │ └── card-01.jpg ← 영문 소문자 + 하이픈만 사용 │ └── README.md ← 프로젝트 소개 (GitHub 첫 화면에 표시)

🚫 절대 경로 사용 금지 이미지나 CSS 경로를 C:/Users/홍길동/Desktop/image.jpg 처럼 내 컴퓨터 경로로 연결하면 GitHub에 올렸을 때 모두 깨집니다. 반드시 images/image.jpg 같은 상대 경로를 사용하세요.

Step 03

GitHub 연동 & 버전 관리

작업 내용을 GitHub에 저장하면 실수로 파일을 삭제해도 복구할 수 있고, 완성 후 무료 배포도 가능합니다.

1
저장소 생성
GitHub에서 새 저장소 만들기
// github.com 접속 후 우측 상단 [+] → New repository Repository name: my-project ← 프로젝트 폴더명과 동일하게 Public 선택 ← GitHub Pages 배포를 위해 필수 [Create repository] 클릭
2
로컬 연결
내 컴퓨터와 GitHub 연결하기
VS Code 터미널 (Ctrl + `) 을 열고 순서대로 입력합니다.
Terminal# 1. Git 초기화 (프로젝트 폴더에서 한 번만) git init # 2. 내 정보 등록 (이 프로젝트에만 적용) git config --local user.name "홍길동" git config --local user.email "email@example.com" # 3. GitHub 저장소 연결 git remote add origin https://github.com/아이디/my-project.git # 4. 첫 번째 저장 및 업로드 git add . git commit -m "첫 번째 커밋: 프로젝트 구조 생성" git push -u origin main
3
매일 반복하는 저장 습관
작업할 때마다 반복하는 3단계
Terminal — 매번 반복# 1. 변경된 파일 추가 git add . # 2. 무엇을 했는지 기록과 함께 저장 git commit -m "헤더 레이아웃 완성" # 3. GitHub에 업로드 git push
💡 커밋 메시지 작성 팁 "수정함", "완료" 대신 무엇을 했는지 구체적으로 적으세요.
"헤더 내비게이션 반응형 처리"  /  "카드 호버 효과 추가"
4
다른 PC에서 시작
다른 컴퓨터에서 프로젝트 받기
공용 PC나 새 컴퓨터에서 작업할 때는 GitHub에서 프로젝트를 내려받아야 합니다.
처음 받을 때는 git clone, 이미 받은 적 있을 때는 git pull을 사용합니다.
Terminal — 처음 받을 때 (clone)# 1. 받고 싶은 폴더로 이동 후 실행 git clone https://github.com/아이디/my-project.git # 2. 생성된 폴더로 이동 cd my-project # 3. 이 PC용 사용자 정보 등록 git config --local user.name "홍길동" git config --local user.email "email@example.com"
Terminal — 이미 clone 한 적 있을 때 (pull)# 최신 상태로 업데이트 git pull
⚠️ 공용 PC 주의사항 작업이 끝나면 반드시 git push로 GitHub에 올린 뒤 자리를 떠나세요.

Q
HTML vs CSS — 작업 타이밍
HTML이 끝난 다음 CSS를 시작하나요?
아닙니다. HTML 전체를 다 완성한 뒤 CSS를 시작하는 방식은 비효율적입니다. 영역 단위로 HTML + CSS를 함께 완성해 나가는 것이 실무 표준입니다.
❌ 오해 — 순서대로
HTML 전체 완성 ↓ CSS 전체 작성 시작 → HTML만 쓰는 동안 결과를 눈으로 확인 불가 → 구조가 맞는지 모른 채 계속 작성하게 됨
✓ 실제 — 영역 단위 반복
헤더 HTML 작성 ↓ 헤더 CSS 작성 ↓ 브라우저 확인 · 수정 ↓ 히어로 HTML 작성 ↓ 히어로 CSS 작성 ↓ 반복 ···
한 영역 작업 시 CSS 작성 순서
CSS/* 1단계 — 레이아웃 구조 먼저 */ header { display: flex; justify-content: space-between; align-items: center; height: 64px; padding: 0 60px; } /* 2단계 — 여백 보정 */ nav { gap: 32px; } /* 3단계 — 색상 */ header { background: #1e1a14; } nav a { color: rgba(255,255,255,0.6); } /* 4단계 — 세부 (폰트, 호버, 그림자 등) */ nav a { transition: color 0.2s; } nav a:hover { color: #fff; }
💡 핵심 정리 "큰 레이아웃 먼저"는 HTML을 다 끝내고 CSS를 시작하라는 뜻이 아닙니다.
각 영역을 작업할 때 세부 스타일보다 구조 CSS를 먼저 잡으라는 원칙입니다.

영역 단위로 HTML → CSS → 브라우저 확인을 반복하면서 완성해 나가세요.

HTML
문서 작성

HTML은 페이지의 뼈대입니다. 스타일(CSS)과 기능(JS)을 붙이기 전에 구조를 먼저 만듭니다. 큰 영역부터 → 작은 요소 순서로 작성하세요.

기본 구조 시맨틱 태그 페이지 연결 이미지 삽입
1
기본 틀
모든 HTML 파일의 기본 구조
모든 페이지에 공통으로 들어가는 틀입니다. ! 입력 후 Tab 키로 자동완성됩니다.
HTML<!DOCTYPE html> <html lang="ko"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>나의 프로젝트</title> <!-- CSS 초기화 — 반드시 가장 먼저 --> <link rel="stylesheet" href="https://meyerweb.com/eric/tools/css/reset/reset.css"> <!-- 구글 폰트 --> <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+KR:wght@400;700&display=swap" rel="stylesheet"> <!-- 공통 CSS --> <link rel="stylesheet" href="css/common.css"> <!-- JS: defer 속성으로 head에서 관리 --> <!-- jQuery를 먼저, 내 스크립트를 나중에 — 순서 중요 --> <script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.7.1/jquery.min.js" defer></script> <script src="js/main.js" defer></script> </head> <body> <!-- 헤더 --> <header> <nav>...</nav> </header> <!-- 메인 콘텐츠 --> <main> <section class="hero">...</section> <section class="card-section">...</section> </main> <!-- 푸터 --> <footer>...</footer> </body> </html>
💡 defer 란? defer 속성을 붙이면 HTML 파싱과 JS 파일 다운로드가 동시에 진행되고, HTML이 모두 로드된 뒤에 JS가 실행됩니다. 덕분에 JS를 <head>에 써도 DOM을 안전하게 조작할 수 있습니다.

선언한 순서대로 실행이 보장되므로 jQuery를 반드시 main.js보다 먼저 작성해야 합니다.
⚠️ async 는 쓰지 말 것 async는 다운로드 완료 즉시 순서 상관없이 실행됩니다. jQuery보다 main.js가 먼저 실행될 수 있어 오류가 납니다. jQuery를 사용할 때는 반드시 defer를 쓰세요.
🚫 웹 표준 — title은 페이지마다 다르게 <title>은 브라우저 탭과 검색 결과에 표시되는 이름입니다. 모든 페이지를 나의 프로젝트로 똑같이 쓰지 마세요.

<title>홈 — 나의 프로젝트</title> ✅
<title>목록 — 나의 프로젝트</title> ✅
<title>나의 프로젝트</title> (모든 페이지 동일) ❌
2
시맨틱 태그
의미 있는 태그 사용하기
태그를 올바르게 사용하면 검색엔진이 페이지를 잘 이해하고, 코드도 읽기 쉬워집니다.
❌ 나쁜 예 — div 남발
<div class="header"> <div class="nav">...</div> </div> <div class="content"> <div class="section"> ... </div> </div> <div class="footer">...</div>
✓ 올바른 예 — 시맨틱
<header> <nav>...</nav> </header> <main> <section class="card-section"> ... </section> </main> <footer>...</footer>
3
Heading 구조
h1 ~ h6 올바르게 사용하기
<h1>은 페이지 전체에 하나만 사용합니다. 제목 단계는 건너뛰지 않고 순서대로 내려가야 합니다.
❌ 나쁜 예

2
레이아웃
Flexbox — 가로 배치의 기본
display: flex를 선언하면 자식 요소들이 가로로 나란히 배치됩니다. 방향·정렬·간격을 속성으로 제어합니다.
css/common.css/* ── Flexbox 기본 구조 ── */ .container { display: flex; /* 자식 요소를 가로로 배치 */ flex-direction: row; /* 기본값 — 가로(row) / 세로(column) */ justify-content: space-between; /* 가로축 정렬 */ align-items: center; /* 세로축 정렬 */ gap: 24px; /* 자식 요소 사이 간격 */ } /* ── 패턴 1 : 헤더 — 로고 좌 / 메뉴 우 ── */ #main-nav { display: flex; justify-content: space-between; /* 양 끝으로 밀어냄 */ align-items: center; padding: 0 60px; height: 64px; } /* ── 패턴 2 : 버튼·태그를 가로로 나열 ── */ .btn-group { display: flex; gap: 8px; /* 버튼 사이 간격 */ flex-wrap: wrap; /* 공간 부족 시 다음 줄로 */ } /* ── 패턴 3 : 카드를 2행 3열로 나열 ── */ /* flex-wrap: wrap + 자식 너비 고정으로 그리드처럼 배치 */ .card-list { display: flex; flex-wrap: wrap; /* 넘치면 다음 줄로 */ gap: 24px; } .card-list > * { /* 한 줄에 3개 — (100% - gap 2개) ÷ 3 */ width: calc((100% - 24px * 2) / 3); }
속성주요 값설명
flex-directionrow / column자식 배치 방향 — 가로 / 세로
justify-contentflex-start / center / space-between / space-around주축(가로) 정렬
align-itemsflex-start / center / flex-end / stretch교차축(세로) 정렬
gappx, rem 등 단위자식 요소 사이 간격
flex-wrapnowrap / wrap줄 바꿈 여부 — wrap 시 다음 줄로 이동
flex숫자 (예: 1)자식 요소의 비율 분배 — flex: 1이면 균등 분배
calc()수식CSS에서 계산식 사용 — calc(100% - 48px) 처럼 단위가 달라도 계산 가능
💡 패턴 3 — calc() 공식 이해하기 calc((100% - gap * (열수 - 1)) / 열수)

3열이고 gap이 24px이면 → calc((100% - 24px * 2) / 3)
2열로 바꾸려면 → calc((100% - 24px * 1) / 2)

gap의 개수 = 열 수 - 1 임을 기억하세요. 3열이면 사이 간격이 2개입니다.

색상 대비 & 포커스 스타일
모든 사용자가 불편 없이 읽을 수 있도록 색상 대비와 키보드 접근성을 지켜야 합니다.
❌ 나쁜 예
/* 낮은 색상 대비 */ body { background: #ffffff; color: #aaaaaa; /* 흰 배경에 회색 → 대비 부족 */ } /* 포커스 스타일 제거 — 절대 금지 */ *:focus { outline: none; }
✓ 올바른 예
/* 충분한 색상 대비 */ body { background: #ffffff; color: #1e1a14; /* 흰 배경에 진한 색 */ } /* 포커스 스타일 유지 또는 커스텀 */ a:focus, button:focus { outline: 2px solid var(--color-primary); }
💡 색상 대비 확인 방법 Chrome 개발자 도구(F12) → Elements 탭 → 텍스트 색상 클릭 → 하단에 대비 비율이 표시됩니다. 4.5 : 1 이상이면 통과입니다.
🚫 outline: none 은 쓰지 말 것 키보드로 탭 이동 시 어느 요소에 포커스가 있는지 표시하는 것이 outline입니다. 이를 제거하면 키보드 사용자가 현재 위치를 알 수 없습니다. 디자인이 마음에 안 들면 none 대신 색상과 두께를 바꿔서 유지하세요.

JavaScript
& jQuery

HTML/CSS로 만든 페이지에 인터랙션을 추가합니다. jQuery를 사용하면 더 간단하게 작성할 수 있습니다.

기본 구조 이벤트 스타일 조작 클래스 조작 시각 효과 종합 예시
1
기본 구조
main.js 기본 틀
js/main.js/** * main.js * jQuery를 사용한 인터랙션 모음 */ // 문서 로드 완료 후 실행 — 모든 코드를 이 안에 작성 $(function () { console.log('스크립트 실행!'); // 여기에 기능들을 추가하세요 });
⚠️ $(function()) 은 여전히 필요한가? defer를 사용하면 HTML 파싱 완료 후 JS가 실행되므로 이론적으로는 생략 가능합니다. 하지만 명시적으로 작성하는 것을 권장합니다. 코드 의도가 명확해지고, 혹시 defer를 빠뜨렸을 때의 안전망이 됩니다.
2
이벤트 연결
이벤트 연결 방법 & 종류
jQuery에서 이벤트는 .on(이벤트명, 함수)로 연결합니다. 하나의 요소에 여러 이벤트를 동시에 연결할 수도 있습니다.
js/main.js// 기본 — 이벤트 하나 연결 $('#btn').on('click', function () { console.log('버튼 클릭!'); }); // 여러 이벤트를 하나의 함수로 연결 (공백으로 구분) $('#input').on('focus blur', function () { console.log('포커스 들어오거나 나감'); });
이벤트발생 시점주요 사용 예
click마우스 클릭버튼, 메뉴, 카드
mouseenter마우스 커서가 요소 안으로 들어올 때호버 효과 시작
mouseleave마우스 커서가 요소 밖으로 나갈 때호버 효과 종료
focus입력 필드에 커서가 들어올 때폼 입력 필드 강조
blur입력 필드에서 커서가 벗어날 때폼 유효성 검사
input입력값이 바뀔 때마다 (실시간)글자 수 카운트, 실시간 검색
change입력값 변경 후 포커스를 벗어날 때select, checkbox, radio
submit폼 제출 버튼 클릭폼 유효성 검사
scroll페이지 또는 요소 스크롤Top 버튼 표시, 헤더 고정
keydown키보드 키를 누를 때Enter 키 제출, 단축키
💡 $(this) 란? 이벤트 핸들러 안에서 $(this)는 이벤트가 발생한 요소 자신을 가리킵니다. 여러 요소에 같은 이벤트를 연결했을 때 클릭된 요소만 조작할 수 있습니다.
3
스타일 조작
.css() — 스타일 직접 변경하기
.css()로 요소의 CSS 속성을 읽거나 직접 변경합니다. 단순한 상태 변경은 클래스 조작(Step 4)을 우선 사용하고, 동적으로 계산된 값을 적용할 때 사용합니다.
js/main.js// 단일 속성 변경 $('#box').css('background-color', '#e07340'); // 여러 속성 동시 변경 — 객체 형태로 작성 $('#box').css({ 'background-color': '#e07340', 'color': '#fff', 'border-radius': '8px' }); // 현재 값 읽기 const color = $('#box').css('color'); console.log(color); // 'rgb(0, 0, 0)' 형태로 반환
⚠️ .css() 보다 클래스 조작을 우선할 것 스타일을 JS에 직접 작성하면 CSS 파일과 분리돼 유지보수가 어려워집니다. 고정된 스타일 변경은 클래스를 미리 CSS에 정의해두고 .addClass() / .removeClass()로 교체하는 방법을 먼저 고려하세요.
4
클래스 조작
addClass / removeClass / toggleClass
CSS에 미리 정의한 클래스를 JS로 추가·제거해서 상태를 변경합니다. 스타일은 CSS에, 동작은 JS에 분리할 수 있어 가장 권장되는 방식입니다.
js/main.js// 클래스 추가 $('#menu').addClass('is-open'); // 클래스 제거 $('#menu').removeClass('is-open'); // 클래스 토글 — 있으면 제거, 없으면 추가 $('#menu').toggleClass('is-open'); // 클래스 보유 여부 확인 — true / false 반환 if ($('#menu').hasClass('is-open')) { console.log('메뉴가 열려있음'); } // 여러 클래스 동시 적용 (공백으로 구분) $('#btn').addClass('active disabled');
💡 클래스 네이밍 관례 JS로 제어하는 상태 클래스는 is-open, is-active, is-hidden 처럼 is- 접두사를 붙이면 CSS 디자인 클래스와 구분하기 쉽습니다.
5
저수준 시각 효과
show / hide / fade / slide
요소를 보이거나 숨길 때 사용하는 메서드입니다. 괄호 안에 밀리초(ms)를 넣으면 애니메이션 속도를 조절할 수 있습니다.
js/main.js// 즉시 표시 / 숨기기 / 토글 $('#box').show(); $('#box').hide(); $('#box').toggle(); // 페이드 — 투명도로 나타나고 사라짐 $('#box').fadeIn(300); // 300ms 동안 나타남 $('#box').fadeOut(200); // 200ms 동안 사라짐 $('#box').fadeToggle(300); // 슬라이드 — 높이로 펼치고 접힘 $('#box').slideDown(300); // 아래로 펼침 $('#box').slideUp(200); // 위로 접힘 $('#box').slideToggle(300); // 완료 후 콜백 — 애니메이션이 끝난 뒤 실행할 코드 $('#box').fadeOut(300, function () { console.log('사라짐 완료'); });
메서드동작애니메이션
show / hide / toggle표시 / 숨김 / 전환즉시 (ms 지정 시 크기+투명도)
fadeIn / fadeOut / fadeToggle나타남 / 사라짐 / 전환투명도(opacity)
slideDown / slideUp / slideToggle펼침 / 접힘 / 전환높이(height)
6
고수준 시각 효과
.animate() — 속성값을 직접 지정해 애니메이션
fade·slide처럼 정해진 동작이 아닌, 원하는 CSS 속성값까지 직접 지정해 애니메이션을 만듭니다. 숫자로 표현되는 CSS 속성이라면 대부분 적용 가능합니다.
js/main.js// 기본 구조 — animate({ 속성: 목표값 }, 속도, 콜백) $('#box').animate({ 'opacity': 0.5, // 투명도 50%로 'margin-top': '20px' // 위 여백 20px로 }, 400); // 페이지 최상단으로 부드럽게 이동 — 가장 많이 쓰이는 패턴 $('html, body').animate({ scrollTop: 0 }, 600); // 특정 요소 위치로 이동 const targetY = $('#section2').offset().top; // 요소의 Y 좌표 $('html, body').animate({ scrollTop: targetY }, 500); // 콜백 — 애니메이션 완료 후 실행 $('#box').animate({ 'width': '200px' }, 300, function () { $(this).addClass('is-expanded'); // 완료 후 클래스 추가 });
구분사용 가능 여부예시
숫자 속성✅ 가능width, height, opacity, margin, padding, top, left, scrollTop
색상 속성❌ 기본 불가color, background-color — jQuery UI 플러그인 필요
transform❌ 불가CSS transition / animation 사용 권장
💡 단순 전환은 CSS transition이 더 적합 버튼 호버, 카드 호버처럼 단순한 스타일 전환은 CSS transition으로 처리하는 것이 성능상 유리합니다. .animate()는 JS로 값을 계산해야 하는 경우(스크롤 이동, 동적 크기 변경 등)에 사용하세요.
7
종합 활용
실제 프로젝트 활용 예시
