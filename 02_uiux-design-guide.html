02
Color System

색상 원칙

🎨
3색 원칙
색을 많이 쓸수록 나빠진다

색상은 최대 3가지만 사용합니다. 주색·배경색·포인트색으로 역할을 나눠서 일관성을 유지하세요.

Primary 주색
버튼, 링크, 강조 텍스트에 사용. 브랜드를 대표하는 색
Background 배경색
전체 배경. 주로 흰색 계열. 콘텐츠가 돋보이게 해야 함
Accent 포인트색
아주 중요한 곳에만 사용. 너무 많이 쓰면 효과 없어짐
⚡
색상 대비 (Contrast Ratio)
가독성의 최소 기준 — WCAG 4.5:1

텍스트와 배경 사이의 밝기 차이가 충분해야 읽을 수 있습니다. 국제 기준(WCAG)은 일반 텍스트 4.5:1 이상, 큰 텍스트는 3:1 이상을 요구합니다.

텍스트 예시
#fff + #767676
✓ 4.5:1 통과
텍스트 예시
#fff + #111
✓ 21:1 우수
텍스트 예시
#fff + #aaa
✗ 2.3:1 실패
텍스트 예시
#ff0 + #fff
✗ 1.1:1 위험
텍스트 예시
짙은배경 + #fff
✓ 9.8:1 우수
텍스트 예시
빨간배경 + 연빨강
✗ 1.5:1 실패
검사 도구: webaim.org/resources/contrastchecker 에서 두 색상을 입력하면 대비율을 바로 확인할 수 있습니다.

04
UX Psychology

UX 원칙 & 심리 법칙

Fitts' Law
클릭 타겟은 크게
클릭 가능한 요소는 충분히 커야 합니다. 작은 버튼은 클릭 미스로 이어져 사용자를 짜증나게 합니다.
모바일 터치 타겟: 최소 44×44px
버튼 padding: 12px 24px 이상
메뉴 항목 높이: 48px 이상
Hick's Law
선택지는 줄여라
선택지가 많아질수록 사용자의 결정 시간이 증가합니다. 메뉴가 너무 많으면 사용자가 포기합니다.
내비게이션: 4~6개 이내
CTA 버튼: 페이지당 1~2개
옵션 많으면 탭·드롭다운으로 숨기기
Jakob's Law
익숙함을 따르라
사용자는 다른 사이트에서 이미 쌓인 경험을 가져옵니다. 창의성보다 익숙한 패턴이 UX에서는 더 좋을 때가 많습니다.
로고는 좌측 상단에
내비게이션은 상단 또는 좌측에
검색은 우측 상단에
📋
닐슨의 사용성 원칙 — 핵심 3가지
Nielsen's Usability Heuristics
1. 시스템 상태 가시성
사용자가 지금 어디에 있는지 항상 알려야 합니다.
현재 페이지 내비게이션 활성 표시 로딩 중 스피너 폼 제출 후 완료 메시지
2. 오류 예방 & 복구
실수를 미리 막고, 실수해도 복구 가능하게 해야 합니다.
폼 유효성 검사 (실시간) 삭제 전 확인 다이얼로그 오류 메시지는 구체적으로
3. 일관성 (Consistency)
같은 기능은 항상 같은 모양·위치에 있어야 합니다.
버튼 색상 전 페이지 통일 헤더·푸터 모든 페이지 동일 버튼 레이블 혼용 금지

📱
Mobile First
모바일을 먼저 만들고 PC로 확장
❌ Desktop First (구식)
PC 먼저 만들고 모바일로 축소
→ 요소가 넘치고 레이아웃 깨짐
→ 불필요한 콘텐츠를 숨기는 데 시간 낭비
→ 모바일 성능 저하
✓ Mobile First (현재 표준)
모바일 먼저 만들고 PC로 확장
→ 핵심 콘텐츠만 남기는 훈련
→ 성능이 자연스럽게 좋아짐
→ 현재 대부분의 트래픽은 모바일
CSS 작성 순서 /* 1. 모바일 (기본 — 작은 화면 먼저) */ .card-grid { grid-template-columns: 1fr; } /* 2. 태블릿 이상에서 확장 */ @media (min-width: 768px) { .card-grid { grid-template-columns: 1fr 1fr; } } /* 3. 데스크톱에서 더 확장 */ @media (min-width: 1024px) { .card-grid { grid-template-columns: repeat(3, 1fr); } }

Common Mistakes

학생들이 가장 많이 하는 실수

# 실수 증상 해결 방법
01 색상 남발 3가지 이상 색이 뒤섞여 정신없음 3색 원칙 주색·배경·포인트 역할 분리
02 여백 부족 요소들이 다닥다닥 붙어 답답함 8pt 배수 "더 비워야 예쁘다"는 원칙 기억
03 폰트 혼용 3가지 이상 폰트로 어수선함 최대 2종 제목용 + 본문용만 사용
04 버튼 묻힘 CTA가 콘텐츠에 묻혀 눈에 안 띔 강한 대비 버튼은 배경과 대비 확실히
05 모바일 미확인 PC에선 예쁜데 모바일에서 깨짐 개발자 도구 F12 → 모바일 뷰 항상 체크
06 정렬 불일치 요소들이 제각각 위치해 어수선함 그리드 정렬 보이지 않는 선에 맞추기
07 작은 클릭 타겟 모바일에서 버튼 클릭 미스 발생 44px 이상 터치 타겟 최소 44×44px
08 낮은 색상 대비 텍스트가 배경에 묻혀 읽기 힘듦 4.5:1 이상 WebAIM 도구로 검사
09 너무 많은 메뉴 내비게이션에 7개 이상 항목 4~6개 이내 나머지는 드롭다운으로
10 호버 효과 없음 클릭 가능한지 알 수 없어 UX 단절 CSS :hover 커서·색상·그림자 변화 추가

Ps 포토샵으로 UI 디자인하기

포토샵은 UI 전용 도구는 아니지만, 학교에서 이미 배웠다면 충분히 목업을 만들 수 있습니다. 파일 생성부터 마크업 전달까지 단계별로 따라해 보세요.

1
새 파일 생성 — 웹 해상도 설정
파일 → 새로 만들기 클릭 후 아래 설정으로 시작합니다.
웹은 72ppi로 설정합니다. 인쇄(300ppi)와 혼동하지 마세요.
/* 데스크톱 기준 */ 너비: 1440px 높이: 900px 해상도: 72 ppi 색상 모드: RGB 8비트 배경 내용: 흰색 또는 투명 /* 모바일 기준 (선택) */ 너비: 375px 높이: 812px
💡 Tip 아트보드를 사용하면 한 파일 안에 PC·모바일 버전을 함께 관리할 수 있습니다.
파일 → 새로 만들기 → 아트보드 체크 후 생성하세요.
2
레이어 그룹 구조 잡기 — HTML 구조처럼
포토샵은 레이어가 많아질수록 관리가 어려워집니다. 처음부터 그룹 폴더로 구조화하면 나중에 수정이 쉽습니다.
/* 레이어 패널 — 권장 구조 */ 📁 header 📁 logo 📁 nav 📁 hero 🖼️ background-image 📁 text-overlay 📁 section-01 ← 이름을 의미 있게! 📁 card-grid 📁 card-01 📁 card-02 📁 footer
⚠️ 주의 레이어 이름을 "레이어 1", "레이어 2"로 남겨두면 나중에 찾을 수 없습니다.
그룹/레이어를 만들자마자 더블클릭 → 이름 변경 습관을 들이세요.
3
가이드 라인 & 그리드 설정
콘텐츠 영역의 좌우 여백을 통일하기 위해 가이드 선을 먼저 설정합니다.
/* 가이드 선 설정 방법 */ 보기 → 새 가이드 레이아웃 열: 12개 여백: 80px (좌우) 간격: 24px (열 사이) /* 또는 수동으로 */ 눈금자에서 캔버스로 드래그 → 가이드 선 생성 보기 → 스냅 → 가이드에 스냅 (ON)
💡 Tip Ctrl+; — 가이드 선 보이기/숨기기 토글
Ctrl+Shift+; — 가이드 선에 스냅 ON/OFF
4
레이아웃 배치 — 헤더부터 순서대로
위에서 아래로, 큰 구조 먼저 잡습니다. 세부 내용보다 영역 구분이 먼저입니다.
/* 작업 순서 */ 1. 사각형 도구(U) → Header 영역 색 블록 2. 사각형 도구(U) → Hero 영역 색 블록 3. 사각형 도구(U) → 카드 블록 반복 배치 4. 문자 도구(T) → 제목, 본문 텍스트 5. 이미지 배치 → 파일 → 포함 항목 가져오기

색상 및 스타일 적용
사각형 레이어를 선택하고 레이어 스타일로 색상·그림자·테두리를 적용합니다.
/* 색상 적용 방법 */ 레이어 더블클릭 → 레이어 스타일 창 → [색상 오버레이] 체크 → HEX 값 입력 /* 스포이드로 색상 추출 */ I 키 (스포이드 도구) → 색상 클릭 → 선택됨 → 전경색 박스 더블클릭 → HEX 복사 /* 그림자 */ 레이어 스타일 → [드롭 새도우] 거리: 4px 퍼짐: 0 크기: 16px 불투명도: 15%
6
수치 추출 — CSS 작성을 위한 값 뽑기
완성된 목업에서 크기·위치·색상을 뽑아 CSS에 적용합니다.
/* 크기·위치 확인 */ 이동 도구(V) → 레이어 클릭 → 상단 옵션 바: W(너비), H(높이), X, Y 확인 → 속성 패널(창 → 속성)에서도 확인 가능 /* 색상 추출 */ 스포이드(I) → 클릭 → 전경색 더블클릭 → HEX 복사 → CSS: color: #4f46e5 /* 텍스트 정보 확인 */ 문자 도구(T) → 텍스트 레이어 클릭 → 문자 패널(창 → 문자): 폰트, 크기, 자간, 행간 확인
💡 Tip 눈금자(Ctrl+R)를 켜두면 요소 사이의 거리를 드래그로 바로 잴 수 있습니다.
7
이미지/아이콘 내보내기
사용할 이미지를 적절한 포맷으로 내보냅니다.
/* 웹용으로 저장 */ 파일 → 내보내기 → 웹용으로 저장 (레거시) 단축키: Ctrl+Alt+Shift+S /* 포맷 선택 기준 */ 사진·배경 이미지 → JPEG (품질 80~90%) 투명 배경 필요 → PNG-24 아이콘·로고 → SVG (일러스트레이터 권장) /* 저장 위치 */ 프로젝트 폴더/images/ 또는 assets/
8
목업 → 마크업 전환
완성된 포토샵 목업을 보면서 HTML/CSS를 작성합니다. 포토샵을 참고 화면으로 띄워두고 코딩합니다.
Ps 포토샵에서 확인한 것

  • 배경색: #1a1a2e
  • 제목 폰트: Noto Sans KR Bold 32px
  • 카드 패딩: 24px
  • 카드 간격: 16px
</> CSS로 변환한 것
.header { height: 60px; background: #1a1a2e; } .title { font-size: 32px; font-weight: 700; } .card { padding: 24px; } .grid { gap: 16px; }

포토샵 vs 피그마 — 언제 뭘 써야 할까?

Ps 포토샵이 유리한 경우
  • 이미지 보정·합성이 필요한 경우
  • 배경 사진을 직접 편집해서 넣을 때
  • 학교에서 이미 포토샵을 배운 경우
  • 복잡한 그래픽 효과(글로우, 텍스처 등)가 필요할 때
◈ 피그마가 유리한 경우
  • UI 전용 작업 (버튼, 카드, 폼 등)
  • 여러 페이지를 한 번에 설계할 때
  • 수치를 정확히 잡고 개발자에게 전달할 때
  • 팀 협업이 필요할 때
  • 무료로 사용하고 싶을 때
Figma · 무료 플랜 기준

◈ 피그마로 UI 디자인하기

피그마(Figma)는 웹 기반 UI 디자인 전용 도구입니다. 무료이고, 설치 없이 브라우저에서 바로 사용할 수 있습니다. 웹/앱 UI 설계에 최적화되어 있어 현업에서도 가장 많이 쓰입니다.

시작 전 준비 figma.com 에서 무료 가입 → 이메일 인증 → 브라우저에서 바로 사용 가능
학교 계메일로 가입하면 Education 플랜(무료)을 신청할 수 있습니다.
1
새 파일 생성 — Frame 설정
피그마에서는 캔버스(Canvas) 위에 Frame을 만들어 페이지 단위로 작업합니다.
/* Frame 생성 방법 */ 1. 단축키 F 누르기 → Frame 도구 선택 2. 우측 패널 → 프리셋 선택: Desktop: 1440 × 1024 iPhone 14: 390 × 844 3. 또는 캔버스에 드래그로 직접 크기 지정 /* 여러 페이지 = 여러 Frame */ Frame을 여러 개 만들어 각각 페이지로 사용 ex) index, about, menu, contact Frame
💡 Tip Ctrl+마우스 휠 — 확대/축소
Space+드래그 — 캔버스 이동
Ctrl+Shift+H — 전체 화면에 맞추기
2
기본 도구 익히기
피그마는 도구가 단순합니다. 이 5가지만 알아도 UI 디자인의 90%를 할 수 있습니다.
V선택 도구 — 요소 클릭·이동
FFrame 도구 — 페이지/컨테이너
RRectangle — 사각형 블록
TText 도구 — 텍스트 입력
IColor Picker — 색상 추출
Ctrl+D요소 복제

Ctrl+Alt+GFrame으로 묶기

피그마만의 기능 요소에 마우스를 올리고 Alt 키를 누르면 다른 요소와의 거리(px)가 빨간선으로 표시됩니다. CSS 수치를 뽑기 매우 편리합니다.
3
Layout Grid — 그리드 설정
그리드는 레이아웃의 뼈대입니다. Frame을 만든 뒤 바로 그리드를 설정하고 그 위에 요소를 배치하면 간격이 일정하게 유지됩니다.

/* 그리드 생성 방법 */ Frame 선택 → 우측 패널 Design 탭 → Layout grid 섹션 [+] 클릭 → 추가된 그리드 아이콘 클릭 → 옵션 팝업 열림 → Type: Column 선택 후 값 입력 /* 데스크톱 12열 권장 설정 */ Count: 12 /* 열 개수 */ Type: Stretch /* Frame 너비에 자동 맞춤 */ Margin: 80 /* 좌우 여백 (px) */ Gutter: 24 /* 열 사이 간격 (px) */ /* 상황별 권장값 */ 데스크톱: Count 12 Margin 80 Gutter 24 태블릿: Count 8 Margin 40 Gutter 16 모바일: Count 4 Margin 16 Gutter 16 /* 그리드 보이기/숨기기 */ Ctrl+G (Windows) / Cmd+G (Mac)
💡 그리드 스타일 저장 — 재사용 Layout grid 우측 스타일 아이콘(점 4개) → [+] → 이름 저장
(예: Desktop/12col)
이후 다른 Frame에서 같은 스타일을 클릭 한 번으로 바로 적용할 수 있습니다.
4
스타일 시스템 — 색상 & 텍스트 스타일 등록
CSS 변수처럼 색상·폰트를 스타일로 등록해두면 전체 디자인을 한 번에 바꿀 수 있습니다.
/* 색상 스타일 등록 */ 우측 패널 Fill 옆 [⊕] 클릭 → 스타일 이름 입력 예: Primary, Background, Text/Body /* 텍스트 스타일 등록 */ 텍스트 선택 → 우측 텍스트 패널 [⊕] 클릭 예: Heading/H1, Body/Regular, Caption /* 활용 */ 다른 요소에 적용: Fill → [스타일 아이콘] → 등록된 스타일 선택 색상 하나 바꾸면 → 그 스타일 쓰는 모든 요소 자동 변경
5
오토 레이아웃 (Auto Layout) — 핵심 기능
피그마의 가장 강력한 기능. CSS의 Flexbox와 동일하게 동작합니다. 요소를 그룹 묶듯 선택하고 Auto Layout을 적용하면 간격이 자동 유지됩니다.
/* Auto Layout 적용 방법 */ 1. 요소 여러 개 선택 2. 우측 패널 → [Auto layout] 버튼 클릭 또는 단축키: Shift+A /* 설정 옵션 (CSS와 대응) */ Direction: Horizontal → flex-direction: row Direction: Vertical → flex-direction: column Gap: 16 → gap: 16px Padding: 24 → padding: 24px Align: Center → align-items: center
💡 왜 Auto Layout을 써야 하나? 텍스트가 길어져도 버튼 크기가 자동으로 늘어나고, 카드를 추가해도 간격이 자동 유지됩니다. 실제 웹의 동작과 가장 가깝게 디자인할 수 있습니다.
6
컴포넌트 — 반복 요소를 재사용하기
버튼, 카드, 헤더처럼 여러 곳에서 반복되는 요소는 컴포넌트로 만들어 두면 한 번 수정으로 전체에 반영됩니다.
/* 컴포넌트 만들기 */ 요소 선택 → 우클릭 → [Create component] 또는 단축키: Ctrl+Alt+K /* 컴포넌트 사용하기 */ Assets 패널(좌측) → 만든 컴포넌트 드래그 → 이것을 Instance(인스턴스)라고 함 /* 수정하면? */ 원본(Main Component) 수정 → 모든 Instance 자동 반영 ✓
💡 추천 컴포넌트 헤더(Nav), 카드, 버튼(Primary·Secondary), 푸터, 입력 필드를 컴포넌트로 만들어 두세요.
7
구글 폰트 연동
피그마는 구글 폰트를 기본 내장하고 있습니다. 검색만 하면 바로 사용할 수 있습니다.
/* 폰트 적용 방법 */ 1. 텍스트 선택 → 우측 패널 폰트 필드 클릭 2. 검색창에 폰트명 입력 예: Noto Sans KR, Black Han Sans 3. 폰트 선택 → 굵기(weight) 선택 4. 크기(px) 및 행간(Line height) 설정 /* CSS로 옮길 때 */ font-family: 'Noto Sans KR', sans-serif; font-size: 16px; line-height: 1.6;
8
Inspect — CSS 수치 자동 추출
피그마의 Inspect 패널에서는 CSS 코드를 자동으로 생성해줍니다. 포토샵과 달리 수치를 직접 계산할 필요가 없습니다.
/* Inspect 사용 방법 */ 1. 요소 클릭 → 우측 상단 [Dev Mode] 탭 선택 (또는 Inspect 탭) 2. 자동으로 CSS 코드 생성됨: /* 피그마가 생성해주는 예시 */ .button { display: flex; padding: 12px 24px; background: #4f46e5; border-radius: 8px; font-size: 16px; font-weight: 600; }
⚠️ 주의 Inspect에서 나온 CSS가 항상 완벽하지는 않습니다. 참고 자료로 활용하되, 실제 코드는 직접 다듬어야 합니다.
9
이미지 내보내기 & 공유
완성된 디자인을 이미지로 내보내거나 링크로 공유합니다.
/* 이미지 내보내기 */ 요소/Frame 선택 → 우측 패널 하단 [Export] 섹션 → [+] 클릭 → 포맷 선택 (PNG / JPG / SVG / PDF) → [Export] 버튼 클릭 /* 배율 설정 */ 1x → 원본 크기 (72ppi) 2x → 레티나 대응 (144ppi) — 보통 이걸 사용 /* 링크 공유 (강사 제출용) */ 우측 상단 [Share] 버튼 → [Copy link] → "링크가 있는 누구나" 설정 → 링크 복사

그리드 생성 방법 — 단계별
1
Frame 선택 — 그리드를 적용할 Frame을 클릭합니다. (페이지 전체 Frame 또는 특정 섹션 Frame 모두 가능)
2
우측 패널 → Design 탭 → Layout grid 섹션 찾기 → [+] 버튼 클릭
3
추가된 그리드의 왼쪽 아이콘(격자 모양)을 클릭 → 그리드 옵션 팝업 열림
4
팝업 상단 드롭다운에서 Grid / Column / Row 중 선택 후 값 입력
5
Ctrl+G (Windows) / Cmd+G (Mac) — 그리드 보이기/숨기기 토글

💡 그리드 스타일 저장 — 여러 Frame에 재사용 한 번 설정한 그리드를 스타일로 저장하면 다른 Frame에 클릭 한 번으로 동일하게 적용할 수 있습니다.

Layout grid 우측 점 4개 아이콘(스타일) 클릭 → [+] → 이름 입력 (예: Desktop/12col)
이후 다른 Frame → Layout grid [+] → 스타일 아이콘 → 저장된 스타일 선택
⚠️ 그리드와 요소 배치 원칙 그리드 선은 가이드일 뿐, 모든 요소가 반드시 열에 딱 맞을 필요는 없습니다.
단, 카드·섹션·버튼 등 주요 요소의 좌우 경계는 컬럼 선에 정렬되어야 균형감이 생깁니다.
